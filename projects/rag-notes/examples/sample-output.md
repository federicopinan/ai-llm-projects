## Core principles

A local-first AI workflow typically follows three principles.

First, **data stays on disk**. Notes, embeddings, and vector indexes live in
files the user owns. Backups are ordinary file copies, not vendor exports.

Second, **inference is local**. A model runs on the user's CPU or GPU, so
prompts never leave the machine. Tools like Ollama and llama.cpp make this
practical on consumer hardware.

Third, **the stack is composable**. Each step — chunking, embedding,
retrieval, generation — can be swapped independently. There is no single
locked-in vendor.

## Practical benefits

- No recurring API costs once the hardware is available.
- Works offline, which is useful on the road or in restricted environments.
- Reproducible behavior: the same model and the same notes always produce the
  same embeddings and answers.
- Easier to audit, since the entire pipeline is open source.

## When it makes sense

Local-first is a strong default for personal knowledge bases, journaling
assistants, and developer tooling. For high-stakes reasoning or large
context windows, a hybrid approach — local retrieval plus an optional cloud

## What changes when Ollama is running

With Ollama running locally (`ollama serve`) and `LLM_MODEL=llama3.1` set,
the `ANSWER` block is replaced by a model-generated paragraph that cites the
chunks above. Example:

```txt
========================================================================
ANSWER
========================================================================

Based on the retrieved notes [local-first-ai.md], local-first AI offers
several benefits: privacy (data stays on disk and prompts never leave the
machine), cost (no recurring API fees), offline operation, reproducible
behavior, and an auditable open-source stack. The notes also flag honest
trade-offs [local-first-ai.md]: smaller local models may underperform cloud
models on hard reasoning tasks, and embedding quality depends on the chosen
model. A hybrid setup — local retrieval with an optional cloud model — is
suggested as a reasonable compromise when reasoning demands are high
[local-first-ai.md].

[generated by Ollama model: llama3.1]
```

The model output varies between runs and between models. The PoC is meant
to be evaluated by inspecting the retrieved chunks, not by trusting the
generated text alone.
